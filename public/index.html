🚀 เริ่มต้นใช้งาน

Claude Code คืออะไร?

AI Agent ที่ทำงานผ่าน Command Line (CLI) ช่วยให้คุณมอบหมายงานเขียนโค้ด, แก้บั๊ก, หรือวางโครงสร้างโปรเจกต์ผ่านการแชท จุดเด่นคือความสามารถในการ "คิด" และวางแผนก่อนลงมือทำ

การตั้งค่าพื้นฐาน

  1. เรียกใช้งาน: เปิดเทอร์มินัลแล้วพิมพ์ claude
  2. เริ่มต้นโปรเจกต์: ใช้คำสั่ง /init เพื่อสแกนโปรเจกต์และสร้างไฟล์ claude.md
  3. โต้ตอบ: เริ่มแชทเพื่อถามคำถามหรือมอบหมายงานได้ทันที

หัวใจของโปรเจกต์: ไฟล์ claude.md

ไฟล์นี้ทำหน้าที่เป็น "หน่วยความจำระยะยาว" ของ Claude Code สำหรับโปรเจกต์นั้นๆ คุณสามารถแก้ไขเพื่อใส่ข้อกำหนดหรือข้อมูลสำคัญที่อยากให้ AI จดจำไว้เสมอ

โหมดวางแผน (Plan Mode)

กด Shift + Tab เพื่อเข้าสู่ Plan Mode ดูแผนการทำงานที่ AI คิดไว้ทีละขั้นตอน ช่วยให้คุณตรวจสอบและอนุมัติก่อนที่โค้ดจะถูกแก้ไขจริง

💡 เวิร์กโฟลว์และเทคนิค

ใช้ Git เป็นจุดเซฟ

ควร Commit การเปลี่ยนแปลงบ่อยๆ เนื่องจาก Claude Code ไม่มีฟังก์ชัน "Undo" ในตัว หากไม่พอใจผลลัพธ์ สามารถใช้ Git ย้อนกลับได้ง่ายดาย

ตรวจสอบงานเสมอ

คิดเสมอว่าโค้ดจาก AI มาจากเพื่อนร่วมทีม ควรตรวจสอบและรีวิวโค้ดทุกครั้งเพื่อคุณภาพและความถูกต้อง

เพิ่มบริบทให้ AI

ยิ่งบริบทมาก ผลลัพธ์ยิ่งแม่นยำ:

  • ลากรูปภาพ: แคปหน้าจอข้อผิดพลาดหรือดีไซน์ที่ต้องการแล้วลากใส่เทอร์มินัล
  • ลากโฟลเดอร์: ลากโฟลเดอร์โปรเจกต์ที่เกี่ยวข้องเพื่อให้ AI เข้าใจภาพรวม
  • ส่ง URL: ส่งลิงก์เอกสาร API หรือหน้าเว็บเพื่อให้ AI ใช้อ้างอิง

⚠️ ข้อจำกัดที่ควรรู้

🔢 Context Limit - ขีดจำกัดบริบท

Claude Code มีข้อจำกัดในการจำข้อมูลที่ส่งให้ในแต่ละเซสชัน:

  • ไฟล์ขนาดใหญ่: หากโปรเจกต์มีไฟล์จำนวนมากหรือไฟล์ใหญ่ AI อาจไม่สามารถดูข้อมูลทั้งหมดได้
  • การสนทนายาว: ยิ่งสนทนายาว AI จะลืมข้อมูลเก่าที่ส่งไปในตอนต้น
  • โค้ดเบสใหญ่: ในโปรเจกต์ขนาดใหญ่ AI อาจไม่เข้าใจความเชื่อมโยงระหว่างไฟล์ทั้งหมด

💡 วิธีแก้: แบ่งงานเป็นชิ้นเล็กๆ ใช้ claude.md จดข้อมูลสำคัญ และเริ่มเซสชันใหม่เมื่อจำเป็น

🎲 Unpredictable Results - ผลลัพธ์ที่คาดเดาไม่ได้

AI อาจให้ผลลัพธ์ที่แตกต่างกันในแต่ละครั้ง แม้จะถามคำถามเดียวกัน:

  • วิธีการต่างกัน: อาจเลือกใช้ Library หรือ Pattern ที่ต่างกันในแต่ละครั้ง
  • คุณภาพไม่สม่ำเสมอ: บางครั้งโค้ดดี บางครั้งมีปัญหา
  • ข้อผิดพลาดซ่อน: โค้ดอาจดูถูกต้องแต่มีบั๊กที่ซ่อนอยู่
  • ตีความผิด: AI อาจเข้าใจความต้องการของคุณผิด

💡 วิธีป้องกัน: ทดสอบโค้ดทุกครั้ง ใช้ Git สำหรับ backup และคอยตรวจสอบความถูกต้องอย่างสม่ำเสมอ

🛡️ Best Practices สำหรับจัดการข้อจำกัด

✅ ควรทำ

  • • ทำงานแบบ Incremental
  • • Test หลังการแก้ไขทุกครั้ง
  • • ใช้ Git Commit บ่อยๆ
  • • อัปเดต claude.md เสมอ

❌ ไม่ควรทำ

  • • ส่งโค้ดจำนวนมากในครั้งเดียว
  • • ไว้ใจผลลัพธ์ 100%
  • • ใช้ในงานที่ Critical โดยไม่ Review
  • • สนทนาต่อเนื่องนานเกินไป

⚡️ Multitasking ด้วย Git Worktrees

แก้ปัญหา Git Conflicts ด้วย Git Worktrees ซึ่งช่วยให้คุณ checkout หลาย branch พร้อมกันในไดเรกทอรีที่แยกจากกัน ทำให้รัน Claude Code หลาย Instance ได้โดยไม่ชนกัน

ขั้นตอนการใช้งาน:

  1. สร้างโฟลเดอร์สำหรับ worktrees นอกโปรเจกต์หลัก
  2. ใช้คำสั่ง git worktree add ../path-to-worktrees/<branch> <branch>
  3. เปิด IDE แยกสำหรับแต่ละ worktree และรัน Claude Code คนละ Instance
ข้อควรระวัง: อย่าสร้าง worktree เป็น subfolder ภายใต้โปรเจกต์หลักเด็ดขาด

⚙️ ควบคุมและปรับแต่งด้วย Hooks

Hooks คือกลไกที่ช่วยให้คุณ "ตั้งโปรแกรม" พฤติกรรมของ Claude Code ได้ ทำให้คุณสามารถควบคุม, สังเกตการณ์, หรือบล็อกคำสั่งที่ไม่ต้องการได้

pre-tool-use

ทำงาน ก่อน รันคำสั่ง (เช่น บล็อกคำสั่งอันตราย)

post-tool-use

ทำงาน หลัง รันคำสั่ง (เช่น Logging)

notification

ทำงานเมื่อ AI ต้องการ Input จากผู้ใช้

stop

ทำงานเมื่อ AI ตอบกลับเสร็จสิ้น

⚡ สร้าง Custom Slash Commands

🎯 Slash Commands คืออะไร?

Slash Commands เป็นคำสั่งพิเศษที่ขึ้นต้นด้วย / ช่วยให้คุณสามารถสร้างคำสั่งแบบกำหนดเองได้ เพื่อเรียกใช้ฟังก์ชันที่ใช้บ่อยๆ หรือรันสคริปต์ที่ซับซ้อนด้วยคำสั่งเดียว

ตัวอย่าง Built-in Commands

/init - สแกนโปรเจกต์และสร้าง claude.md
/help - แสดงคำสั่งที่ใช้ได้
/reset - เริ่มเซสชันใหม่

🔧 การสร้าง Custom Commands

1. สร้างไฟล์ Configuration

สร้างไฟล์ claude_config.json ในโฟลเดอร์ของโปรเจกต์:

{
  "customCommands": {
    "/build": {
      "description": "Build and test the project",
      "command": "npm run build && npm test"
    },
    "/deploy": {
      "description": "Deploy to production",
      "command": "npm run build && git add . && git commit -m 'Deploy' && git push origin main"
    },
    "/setup": {
      "description": "Install dependencies and setup environment",
      "command": "npm install && cp .env.example .env && npm run db:migrate"
    }
  }
}

2. คำสั่งที่ซับซ้อนขึ้น

คุณสามารถสร้างคำสั่งที่รับ parameter ได้:

{
  "customCommands": {
    "/component": {
      "description": "Create a new React component",
      "command": "mkdir -p src/components/{{name}} && echo 'import React from \"react\";\\n\\nconst {{name}} = () => {\\n  return 
{{name}}
;\\n};\\n\\nexport default {{name}};' > src/components/{{name}}/{{name}}.tsx"
    },
    "/test": {
      "description": "Run specific test file",
      "command": "npm test -- --testNamePattern={{pattern}}"
    }
  }
}

📝 Advanced Configuration

Multi-step Commands

{
  "customCommands": {
    "/release": {
      "description": "Full release process",
      "steps": [
        "npm run test",

        "npm version patch",
        "git push origin main --tags",

  }
}

💾 Save / Resume - การจัดการบริบท

🎯 ทำไมต้อง Save/Resume?

เนื่องจาก Claude Code มีข้อจำกัดเรื่อง Context Limit การบันทึกและกู้คืนงานจึงเป็นกลยุทธ์สำคัญ ที่ช่วยให้คุณทำงานต่อเนื่องโดยไม่สูญเสียความก้าวหน้า

💡 หลักการสำคัญ: ถือว่า Save/Resume เป็นส่วนหนึ่งของ Development Methodology ไม่ใช่แค่การแก้ปัญหาชั่วคราว

⚡ คำสั่งพื้นฐาน

บันทึกความก้าวหน้า

# บันทึกสถานะปัจจุบัน
claude save progress.md "current status and next steps"

# บันทึกโค้ด
claude save filename.py "code content here"

# บันทึกแผนงาน
claude save plan.md "project requirements and approach"

กู้คืนงาน

# กู้คืนจากไฟล์
claude resume progress.md

# กู้คืนพร้อมข้อมูลเพิ่มเติม
claude "Continue working on X. Previous work is in filename.py"

📋 Templates สำคัญ

1. Project Status File

claude save status.md "
# Project: [ชื่อโปรเจกต์]
## Done: auth, database, API routes
## Current: order processing  
## Next: validation, error handling
## Files: auth.py, models.py, routes.py
"

2. Checkpoint Template

claude save checkpoint.md "
## Progress: User auth complete
## Current: Working on cart feature
## Files: auth.py, cart_models.py  
## Next: cart API endpoints
## Issues: [ปัญหาที่พบ]
"

3. Handoff Template

claude save handoff.md "
## Task: [งานปัจจุบัน]
## Progress: [สิ่งที่ทำเสร็จแล้ว]
## Next: [ขั้นตอนต่อไป]
## Files: [ไฟล์ที่เกี่ยวข้อง]
## Notes: [ข้อมูลสำคัญ]
"

🏗️ Modular Organization Strategy

แบ่งงานออกเป็นส่วนๆ และบันทึกแยกกัน:

# บันทึกแยกตาม Component
claude save models.py "User, Product models"
claude save routes.py "API endpoints"  
claude save utils.py "Helper functions"
claude save tests.py "Unit tests"

# บันทึกแยกตาม Feature
claude save auth_feature.md "Authentication system complete"
claude save cart_feature.md "Shopping cart in progress"

🔄 Common Workflow

ขั้นตอนการทำงาน

  1. เริ่มด้วยแผน: claude save plan.md "Feature requirements"
  2. บันทึกทีละ Component: claude save models.py "Data models"
  3. สร้าง Checkpoint: claude save checkpoint.md "Models done"
  4. Resume เซสชันใหม่: claude resume checkpoint.md
  5. อ้างอิงไฟล์: claude "Continue API. Models in models.py"

💡 Best Practices

✅ ควรทำ

  • • Save เร็ว - อย่ารอให้ Context เต็ม
  • • ใช้ชื่อไฟล์ที่ชัดเจน
  • • ใส่ Context ให้ครบ
  • • สร้าง Checkpoint สม่ำเสมอ
  • • อ้างอิงไฟล์เมื่อ Resume

❌ ไม่ควรทำ

  • • รอจนถึง Context Limit
  • • ใช้ชื่อไฟล์ที่คลุมเครือ
  • • บันทึกแค่โค้ดไม่มี Context
  • • ลืม Save ความก้าวหน้า
  • • Resume โดยไม่อธิบายงานเก่า

🚨 Emergency Recovery

เมื่อหลงทางหรือลืมงานที่ทำ:

# กู้คืนสถานะล่าสุด
claude resume status.md
claude resume latest_checkpoint.md

# หาไฟล์ที่เกี่ยวข้อง
claude "List all .md files and their contents"
claude "Show me the latest progress files"

💡 เทคนิค: ตั้งชื่อไฟล์แบบ timestamp เช่น 2024-01-15_checkpoint.md เพื่อง่ายต่อการติดตาม

🔌 MCP (Model Context Protocol)

MCP คืออะไร?

Model Context Protocol (MCP) เป็นโปรโตคอลที่ช่วยให้ Claude Code สามารถเชื่อมต่อกับเครื่องมือและบริการภายนอกได้ ขยายความสามารถของ AI ให้ทำงานกับระบบต่างๆ ได้อย่างมีประสิทธิภาพ

MCP Services แนะนำ

🖥️ Desktop Commander

การเชื่อมต่อ MCP

MCP ช่วยให้ Claude Code มีความสามารถเพิ่มขึ้นอย่างมาก:

  • ขยายฟังก์ชัน: เพิ่มเครื่องมือใหม่ๆ โดยไม่ต้องแก้ Core System
  • ทำงานข้ามระบบ: เชื่อมต่อกับ Database, API, หรือ Cloud Services
  • Automation: ทำงานอัตโนมัติที่ซับซ้อนข้ามหลายระบบ
  • ปรับแต่งได้: สร้าง MCP Server เฉพาะสำหรับความต้องการของทีม

The Big Three

หัวใจสำคัญของ AI Coding ยังคงเป็น 3 สิ่งนี้เสมอ: บริบท (Context), โมเดล (Model), และ พรอมต์ (Prompt). การเข้าใจหลักการเหล่านี้จะช่วยให้คุณปรับตัวเข้ากับเครื่องมือ AI ใหม่ๆ ได้เสมอ

อนาคต
